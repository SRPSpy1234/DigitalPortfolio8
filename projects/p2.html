Project2_1: DC Motor

2/6/26

//integer variable storing switch/button state initially set to 0/off
int switchState = 0;

//constant variable assigning button/switch to input pin #
const int buttonPin = 2;

//constant variable assigning motor to output pin #
const int motorPin = 9;

//setup function
void setup(){

//function that will set pin for button as INPUT
pinMode(buttonPin, INPUT); 
//function that will set pin for motor as OUTPUT
pinMode(motorPin, OUTPUT);

} 

void loop() {
	//read the state of the button and store in int switchState variable
switchState = digitalRead(buttonPin); 

//turn the motor on/HIGH or off/LOW
if(switchState == HIGH) {
//turn the motor on
digitalWrite(motorPin, HIGH);
}

else {

//turn the motor off
digitalWrite(motorPin, LOW);
		
} 
}

Project Description

For the code portion of 2_1, we were provided with a skeleton code and had to fill in the blanks. We had to create our own variables and chose what pins to use for the buttons and motor. The code checks the state of the button and if it's pressed, it turns the motor on. If the button is not pressed, the motor is off. The difficulty I encountered was before we were supplied a code skeleton, I created a code that simply just turns the motor on and off every 2 seconds. For project materials, we used a DC motor, a button, and an Arduino to create a simple circuit. The button acts as a switch and when pressed, it completes the circuit and powers the motor. We used 2 9v batteries in this project. 1 battery powers the Arduino and the other power the motor. This is done so we have enough power to run both with enough speed. For some of the code, we used new functions and variables that we hadn't before. One of the ones we used was constant variables: it's a variable that is read only and cannot be changed after it's been assigned. The pinMode command is used to set a specific pin as input or output and we used it to set the button pin as input and the motor pin as output. The variable we used for switchState is an integer variable that keeps the state of the button (pressed or not pressed) and we used it to find out if the motor should be on or off. The difference between digitalWrite and digitalRead is that digitalWrite sends a HIGH or LOW signal while digitalRead reads the signal from a pin and stores it as a variable. For comparison operators, we used "==" to check the pressed state of the button and if it registered as HIGH, the motor would power on Some of the real life applications of this project include using a button to control a motor in something like a toy car. It can also be used in something like a fan but with a switch instead of a button.

Project2_2: Potentiometer

2/12/26

//constant variable assigning potentiometer to the analog pin
//set to const because value won't change
const int potPin = A1;

//constant variable assigning motor to output pin #
const int motorPin = 9;

//setup function
void setup(){

//function that will set pin for button as INPUT
pinMode(potPin, INPUT); 
//function that will set pin for motor as OUTPUT
pinMode(motorPin, OUTPUT);

} 

void loop() {
//creates a variable to store the value of the potentiometer
int potVal = analogRead(A1);

//variable to store the potentiometer value to motorSpeed
//mapping the function that executes the conversion
int motorSpeed = map(potVal, 0, 1023, 0, 255);

analogWrite(motorPin, motorSpeed);

//ten millisecond delay to prevent fluctuations in motor speed
delay(10);
}
